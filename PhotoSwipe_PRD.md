> 版本：v1.0

> 技术栈建议：React Native（iOS + Android 双端）或 Flutter

## 一、产品概述

### 1.1 产品定位

PhotoSwipe 是一款以「短视频流」为交互范式的手机相册整理工具。用户通过上下滑动浏览随机抽取的照片流，向左滑删除、向右滑收藏，快速完成相册整理，并通过统计模块获得成就感反馈。

### 1.2 核心价值

- 降低整理门槛：沉浸式全屏浏览，无需进入传统相册页面
- 决策流畅：左删右藏，手势直觉化，单张照片决策时间 < 1 秒
- 正向激励：连续整理天数、已释放空间等数据增强用户动力
- 安全兜底：删除先进废纸篓，30 天内可恢复，避免误操作损失

### 1.3 目标平台

- iOS 16+
- Android 10+

## 三、页面详细设计

### 3.1 图像流页（主页）

#### 布局

- 全屏沉浸式，照片占满整个屏幕（含安全区）
- 顶部状态栏：半透明黑色蒙层，显示系统时间 + 信号图标
- 右侧进度条：竖向小点，指示当前位置（类似 Stories 进度）
- 底部信息层：渐变蒙层上显示拍摄日期、地点、当前张数
- 底部导航栏：半透明黑色背景，四 Tab

#### 核心交互——手势系统

| 手势 | 方向 | 行为 | 视觉反馈 |
|------|------|------|----------|
| 上滑 | ↑ | 切换到下一张 | 照片向上滑出，新照片从下进入 |
| 下滑 | ↓ | 返回上一张 | 照片向下滑出，上一张从上进入 |
| 左滑 | ← | 删除照片，移入废纸篓 | 界面叠加红色蒙层，左侧出现「删除」徽标，照片向左飞出 |
| 右滑 | → | 收藏照片，加入收藏夹 | 界面叠加绿色蒙层，右侧出现「收藏」徽标，照片向右飞出 |
| 点击 | — | 进入照片查看页（全屏查看模式） | 照片展开动画 |

#### 手势触发阈值

- 左右滑动：超过屏幕宽度 30% 时触发操作，否则回弹
- 上下滑动：超过屏幕高度 20% 时触发切换

#### 操作徽标样式

- 删除徽标：红色半透明胶囊，含垃圾桶图标 + 「删除」文字，出现在画面左侧中央
- 收藏徽标：绿色半透明胶囊，含心形图标 + 「收藏」文字，出现在画面右侧中央
- 徽标透明度与滑动距离成正比（线性插值）

#### 照片随机算法要求

- 从系统相册随机抽取，避免短期内重复出现同一张
- 优先展示距上次查看时间最长的照片（「久违照片优先」策略）
- 排除已删除（废纸篓内）和已收藏的照片（可在设置中关闭）
- 每次进入 App 预加载 10 张，滚动至第 7 张时继续加载

#### 底部信息层内容

```
拍摄日期 · 当前第 N 张
📍 拍摄地点（如有 GPS 信息）
```

### 3.2 照片查看页（全屏查看模式）

从图像流页点击照片进入，或在图像流中双击进入。

#### 布局

- 全屏黑色背景，照片居中显示
- 顶部操作栏（半透明）：返回按钮 / 当前位置「12/348」/ 更多菜单
- 底部操作栏（半透明）：删除、分享、收藏、详情 四个操作按钮
- 放大状态下顶部/底部操作栏自动隐藏，单击屏幕重新显示

#### 手势系统（查看模式）

| 手势 | 触发条件 | 行为 |
|------|----------|------|
| 双击 | 任意缩放状态 | 当前为 1× → 放大至 2×（以点击位置为中心）；当前为 2× → 还原至 1× |
| 双指捏合/张开 | 任意 | 自由缩放，范围限制 1× ~ 5× |
| 单指拖拽 | 缩放 > 1× | 平移照片，超出边界自动回弹 |
| 单指拖拽 | 缩放 = 1× | 左右滑动切换上/下一张；下拉退出查看页（带缩小关闭动画） |
| 下拉 | 缩放 = 1× | 退出查看页，返回图像流，带「缩小消失」过渡动画 |

#### 手势优先级（冲突处理）

放大状态（> 1×）时：平移手势 > 切换图片手势（左右滑动被禁用）。  
1× 状态时：切换图片手势 > 平移。

#### 缩放交互细节

- 双击放大以点击位置为中心，确保点击区域出现在视口中央
- 捏合缩放超过 5× 时有阻力感（rubber band），松手弹回 5×
- 捏合缩小低于 1× 时有阻力感，松手弹回 1×
- 缩放比例指示器：放大时屏幕中央短暂显示「2×」「3.5×」等提示，1.5 秒后淡出

#### 照片详情抽屉

从底部「详情」按钮上拉展开，显示：

- 文件名、拍摄时间（精确到秒）
- 设备型号、拍摄参数（光圈、快门、ISO）
- GPS 坐标 + 地图缩略图（如有）
- 文件大小、分辨率、格式

## 四、数据模型

### 4.1 照片记录（PhotoRecord）

```typescript
interface PhotoRecord {
  id: string                    // 系统相册唯一标识
  uri: string                   // 本地文件路径
  createdAt: Date               // 拍摄时间
  lastViewedAt: Date | null     // 上次在 App 内查看时间
  status: 'pending' | 'kept' | 'deleted' | 'favorited'
  location?: {
    latitude: number
    longitude: number
    placeName: string
  }
  fileSize: number              // 字节
  width: number
  height: number
}
```

### 4.2 收藏集（FavoriteCollection）

```typescript
interface FavoriteCollection {
  id: string
  name: string
  type: 'location' | 'person' | 'time' | 'custom'
  coverPhotoId: string
  photoIds: string[]
  createdAt: Date
  updatedAt: Date
}
```

### 4.3 整理会话（SortingSession）

```typescript
interface SortingSession {
  id: string
  date: Date                    // 整理日期（精确到天）
  viewedCount: number           // 查看张数
  favoritedCount: number        // 收藏张数
  deletedCount: number          // 删除张数
  freedSpace: number            // 释放空间（字节）
  durationSeconds: number       // 整理时长
}
```

### 4.4 废纸篓记录（TrashRecord）

```typescript
interface TrashRecord {
  photoId: string
  deletedAt: Date
  expiresAt: Date               // 自动永久删除时间（deletedAt + 30天）
}
```

## 六、动画与过渡规范

| 场景 | 动画类型 | 时长 | 曲线 |
|------|----------|------|------|
| 图像流上下切换 | 位移 + 弹簧 | 300ms | spring(damping: 0.8) |
| 左右滑动删除/收藏 | 位移 + 透明度 | 250ms | ease-out |
| 回弹（未触发阈值） | 弹簧回位 | 200ms | spring(damping: 0.6) |
| 照片查看页进入 | 展开放大 | 280ms | ease-in-out |
| 照片查看页退出（下拉） | 缩小淡出 | 220ms | ease-in |
| 双击放大 | 缩放 | 200ms | ease-out |
| 缩放指示器显示/消失 | 透明度 | 淡入 150ms / 淡出 300ms | ease |
| 操作徽标（左右滑时） | 透明度（跟随手指线性） | 实时 | linear |

## 七、边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 相册为空 | 显示空状态页，引导用户授权或说明已全部整理 |
| 所有照片已整理完毕 | 显示「全部整理完毕」庆祝页，展示本次统计数据 |
| 误操作删除 | 底部弹出 Toast「已删除，点击撤销」，5 秒内可撤销 |
| 误操作收藏 | 底部弹出 Toast「已收藏，点击撤销」，5 秒内可撤销 |
| 无 GPS 信息的照片 | 地点字段不显示，收藏夹「地点」分组中不纳入该照片 |
| 存储空间不足（废纸篓满） | 提示用户清空废纸篓 |
| 放大状态下左右滑动 | 忽略切换手势，仅执行平移 |
| 网络断开 | 所有功能正常（纯本地操作），无需网络 |

## 八、非功能性需求

### 8.1 性能

- 图像流切换帧率：稳定 60fps
- 照片加载：当前照片显示 < 100ms，预加载 ±3 张
- 首屏启动时间：< 2 秒（冷启动）

### 8.2 存储

- App 本身不存储照片副本，所有操作直接操作系统相册
- 整理记录（SortingSession）存储在本地数据库（SQLite / Realm）
- 废纸篓逻辑：照片保留在系统相册「最近删除」，App 仅记录删除时间

### 8.3 隐私

- 所有数据本地处理，不上传服务器
- 人脸识别在设备端完成，不联网
- 「整理建议」的截图检测基于文件名/尺寸规则，不需要云端 AI

## 十、给 Claude Code 的补充说明

1. 手势库推荐：React Native 使用 `react-native-gesture-handler` + `react-native-reanimated`；Flutter 使用内置 `GestureDetector` + `InteractiveViewer`

2. 照片访问：React Native 使用 `@react-native-camera-roll/camera-roll`；Flutter 使用 `photo_manager`

3. 缩放组件：建议封装一个独立的 `ZoomablePhoto` 组件，内部处理所有缩放/平移逻辑，对外只暴露 `onDoubleTap`、`onDismiss` 等回调

4. 手势冲突解决：上下切换手势（图像流）与照片查看页的下拉退出手势需要通过父子组件手势传递（`simultaneousHandlers` / `waitFor`）仔细处理，避免冲突

5. 图像流性能：使用虚拟列表（`FlatList` / `ListView`），只渲染当前及前后各 2 张，避免内存堆积

6. 颜色规范：
   - 主题色（紫色）：`#7F77DD`
   - 删除（红色）：`#E24B4A`，蒙层：`rgba(200, 40, 40, 0.12)`
   - 收藏（绿色）：`#639922`，蒙层：`rgba(40, 180, 100, 0.12)`
   - 背景（深色）：`#000000`（照片区）/ `rgba(0,0,0,0.55)`（操作栏）
